Component library Accordion

The accordion component lets users show and hide sections of related content on a page.

View more general guidelines on this component in the GOV.UK design system.

The accordion is primarily is used primarily on view latest funding and MYESF allocation statements.

JavaScript

To make the accordion functional, you need to reference the GOV.UK Design System JavaScript file named accordion.js.

You will then need to add a reference to this file on any HTML pages which need an accordion.

When to use this component

This component allows for users to expand and hide content. Use this component when:

  • there is a need to reveal information in stages
  • you have multiple sections which are related

When not to use this component

Accordions hide information from the user, we must be sure that users will be able to find what they are looking for. We must not:

  • hide information that is related to more than one section of the page
  • hide any filters, questions, navigational controls

We must also be careful that users are not confused by how the accordion works. We should not use them on the same page as other filtering components such as details and tabs.

Accordions, tabs and details

Our version of the accordion can be customised to meet the user's content needs. The expanded section of the accordion is flexible and can contain other components such as icons, tables, tags etc.

How it works

The accordion component uses JavaScript. When JavaScript is not available, users will see all the content displayed with the section labels as headings.

Accordions with additional information

There are some cases where accordions need to display more information than the standard pattern found on view latest funding.

In MYESF allocation statements (parent view), there is a need to display information such as: organisation name, status tag and variance.

These do not need to be used all together but the example below shows how each can be adopted.

JavaScript

To make the accordion functional, you need to reference the GOV.UK Design System JavaScript file named accordion.js.

You will then need to add a reference to this file on any HTML pages which need an accordion.

The icons in this component use a third party JavaScript library to dynamically generate the optimum CSS for the device utilising it. You will need to reference the following JavaScript resource below:

<script src="https://kit.fontawesome.com/71943611db.js" crossorigin="anonymous"></script>

Find out how your users interact with the content

Users might need the sections they have opened to stay open if they leave and then return to the page.

We have configured the accordion to remember the users selections within the same session within our service. This can be changed if there are valid user needs for other behaviours elsewhere.

Research on this component

Users observed encountering this component found the 'plus' and 'minus' icons familiar, and understood this to mean they could retrieve more information upon clicking.

There is evidence to suggest that the accordion is interpreted as a way to access more granular data, for example, the by school breakdown of a local authority's dedicated schools grant allocation.

Users have also expressed that the component achieves their desire to see valuable information within a single page, rather than having to visit a separate location to gain context for a headline figure.
