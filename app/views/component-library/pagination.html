Component library Pagination

Helps users navigate through multiple pages. Typically used for pages of search results.

This component is not currently in the GOV.UK design system, but is in their GitHub backlog.

Pagination is used across all of our services, examples can be found on MYESF document exchange, view latest funding and view national funding formula for schools.

JavaScript

To offer users a dynamic client side pagination experience, you can reference this example JavaScript file named dynamic-pagination-for-divs.js.

You will then need to add a reference to this file on any HTML pages which need pagination. You will also need to pass in 5 mandatory parameters, as shown in the example below:

$(document).ready(function () {

  dynamicPagination(
    // Total items (totalListCount)
    52,
    // Boolean whether to clear down the session data or not (clearLocalStorage)
    true,
    // How many items to show per page (itemsPerPage)
    25,
    // What to name the thing (singular) being displayed (itemTypeLabel)
    "result",
    // What to name the things (plural) being displayed (itemTypeLabelPlural)
    "results"
  );

});

When to use this component

Pagination is typically used on pages to help navigate through long lists of items, documents or search results.

When not to use this component

Do not use this component to split up steps of a task or user journey.

Do not use pagination on statement pages with different types of data like in view latest funding. This is because page numbers do not help users to distinguish between different types of data.

How it works

We have set rules on pagination across our services. Pagination should appear on:

  • search results pages after 25 results (e.g. view latest funding)
  • document pages after 10 results (e.g. MYESF document exchange)

Client side scripting

We recommend using client side scripting (e.g. JavaScript) to make this component more dynamic for users. However, this should be implemented as a progressive enhancement method - not the only way the pagination component works.

This component should:

  • work dynamically client side if JavaScript is enabled in the user browser
  • fall back to use server side processing if JavaScript is not enabled

Research on this component

As long as this component is made visible to users at the bottom of the initial results screen, there have been few issues with users recognising there are additional results pages, and subsequently clicking through.

There is limited data available on the differentiation users make between results and documents when making use of the pagination component, but in the cases this has been observed, these were interpreted by users in the same way.
