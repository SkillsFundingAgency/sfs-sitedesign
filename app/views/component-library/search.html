Help users search through large collections of information, organisations or documents using a search box component.

This component is not currently in the GOV.UK design system, but is in their GitHub backlog.

The search component is used across most of our services. It can be found in MYESF, view latest funding and view national funding formula for schools.

When to use this component

The search component is used across most of our services. It helps users find information about a school, college or other organisation. It is also used to help users select an option from a large list of choices. For example, reducing a large list of radio buttons.

How it works

Implementation rules

Ensure that:

  • all of the search component types start by offering all users a server side processing method
  • offer client side scripting as a progressive enhancement method
  • when client side scripting is not enabled, a clickable button is available for the user to search with
  • error handling is made available to help users find what they are looking for (e.g. such as preventing empty searches

Search for an entity

Allowing users to search from a large collection of organisations. It can make a user’s journey quicker and easier if done correctly.

Make sure the search criteria matches what users would search for. For example, only allowing a user to search by URN is not helpful when they only know a school's name.

A ‘search results’ page is shown to the user when we find more than one possible match for their search. They can then use the filter and the details displayed alongside the results to make a choice.

If a search does not return any results, we show the user a ‘no results’ page. This will help the user correct their search terms or explain why we do not have what they are looking for.

If a search term matches exactly, the user is sent straight to the relevant page.

Search a large list of choices

A search can be used to find something in a long list of items, such as a list of academies.

When a user searches for a term, the list below will automatically refresh to match the words in the search box.

Provide a ‘clear filters’ option to allow users to quickly unselect all filters.

JavaScript

To offer users a dynamic client side experience for searching, you can reference the following example JavaScript files named:

  • checkbox-editing.js
  • search-radio-buttons.js

You will then need to add a reference to each of the files on any HTML pages which need this functionality.

Client side scripting

We recommend using client side scripting (e.g. JavaScript) to make this component more dynamic for users. However, this should be implemented as a progressive enhancement method - not the only way this component works.

This component should:

  • work dynamically client side if JavaScript is enabled in the user browser
  • fall back to use server side processing if JavaScript is not enabled

Research on this component

In the main, users understand the search component when it is presented, especially with the addition of a 'search' icon. It has been observed that some users confuse search boxes within a service with the wider gov.uk search box if they exist on the same page. Labelling the component clearly to characterise the collection it performs a search on has helped to dispel such confusion during past testing.
