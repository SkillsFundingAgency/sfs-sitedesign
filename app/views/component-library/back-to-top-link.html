Component library Back to top link

A link to help users quickly navigate back to the top of pages when they reach the bottom.

This component is not currently in the GOV.UK design system, but is in their GitHub backlog.

The back to top link is used across all of our services, an example can be found on MYESF document exchange.

When to use this component

This component should be used to help users navigate back to the top of long pages.

When not to use this component

Do not use this component anywhere other than the bottom of the page.

How it works

This component can be dynamically hidden or shown on pages where list items or content length can change. Across our services, the back to top link should be shown:

  • any time that pagination is triggered
  • if there are more than 25 listed items on any page where pagination is not recommended (e.g. view latest funding)

With pagination

Use this type of the component:

  • whenever pagination is triggered on a page
  • in conjunction with the pagination component

It should sit below the pagination section on the right hand side, like in the below example.

JavaScript

To offer users a dynamic client side pagination experience, you can reference this example JavaScript file named dynamic-pagination-for-divs.js.

You will then need to add a reference to this file on any HTML pages which need pagination. You will also need to pass in 5 mandatory parameters, as shown in the example below:

$(document).ready(function () {
  
    dynamicPagination(
      // Total items (totalListCount)
      52,
      // Boolean whether to clear down the session data or not (clearLocalStorage)
      true,
      // How many items to show per page (itemsPerPage)
      25,
      // What to name the thing (singular) being displayed (itemTypeLabel)
      "result",
      // What to name the things (plural) being displayed (itemTypeLabelPlural)
      "results"
    );
  
  });

Stand alone

There are some instances when pagination is not required on a page, but having a back to top link is. For example, if users are viewing a funding statement page (e.g. view latest funding) then we do not trigger pagination but still provide users with a back to top link. If any page has more than 25 listed items on it, then users will still need a back to top link.

Research on this component

Users have utilised this link on numerous occasions during testing when navigating back to the top of the page.

Excessive scrolling has been identified by past research participants as a frustration and obstacle to retrieving data or continuing their journey.
