Component library Filter a list

A way of reducing items on a page to help a user find what they are looking for, or make a comparison.

This component is not currently in the GOV.UK design system, but is on their GitHub backlog.

The filter component is used across all of our services, examples can be found on MYESF document exchange, MYESF allocation statements and view national funding formula for schools.

JavaScript
Warning We recommend creating or finding your own code to achieve this functionality.

The client side scripting required to make this component work is currently being updated in the prototype (filters-and-checkboxes.js), to ensure that:

  • filter sections work in combination
  • filters should automatically update the list of results when a filter is selected
  • screen readers can announce which filter has been selected and how many results are showing on the page after filtering

When to use this component

Use this component on pages with long lists of items. This helps users reduce the amount of scrolling and scanning, like on search results pages.

Use this component when users may need to compare multiple related items, like comparing two funding types.

When not to use this component

Do not use filters on the same page as other filtering methods, such as accordions or tabs.

How it works

Implementation rules

Ensure that:

  • filters are unselected by default when a page loads
  • users can have and use multiple filters at once to reduce content
  • all filters work in combination not independently (e.g. show me all matches from selected filter types 'status' and 'organisation')
  • when a user selects any filter they are given accessible feedback in their browser
  • users must have the ability to clear all filters at once with a "clear filters" link

Standard - less than 25 filter options

Use this type of the component when users have less than 25 options to choose from in a single filter section category.

This is the default design type of this component to use for displaying funding totals figures to end users in MYESF. Use this type whenever possible for displaying funding totals to users in the service.

When there are greater than 25 options in a single filter section category, users need to be able to search this list so they can more easily find what they need. This should work client side with JavaScript, but must also work server side to cater for all user configurations.

JavaScript

To offer users a dynamic client side searchable filter experience, you can reference this example JavaScript file named search-filter-options.js.

You will then need to add a reference to this file on any HTML pages which need this functionality.

Client side scripting

We recommend using client side scripting (e.g. JavaScript) to make this component more dynamic for users. However, this should be implemented as a progressive enhancement method - not the only way this component works.

When client side scripting is enabled:

  • filters should automatically update the list of results when a filter is selected
  • users must be able to open or close any filter section easily with any input (e.g. via keyboard)
  • when there are greater than 25 options in a single filter section category, allow users to search that list to reduce their options instantly

When there is no enabled client side scripting:

  • offer users a button labelled "update results" so they can still filter results via server side scripting
  • long lists of filter options (e.g. greater than 25 options) will be scrollable but not searchable

Accessibility

Our filters have been built so that screen readers can announce which filter has been selected and how many results are showing on the page after filtering.

Research on this component

Users in general find this feature useful when narrowing down a larger result list and identifying the correct link.

There have been comments from users that they anticipate that when another filter is selected, a previous active filter should be cleared.
