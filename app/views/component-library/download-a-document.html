Component library Download a document

A recognisable icon, accompanied by a file link and supporting content. Users can use it to download a file to their local drive.

This component is not in the GOV.UK design system, but is in their GitHub backlog.

The download component is used across all of our services. It can be found on view latest funding.

JavaScript 
$(document).ready(function () {

  $(".expand-hidden-section").click(function (e) {

    e.preventDefault();

    var expandHiddenSectionState = $(this).attr('aria-expanded');
    var hiddenSection = $(this).attr('href');

    if (expandHiddenSectionState == "false") {
      $(this).attr("data-expanded", true);
      $(this).attr("aria-expanded", true);
      $(hiddenSection).show();
      $(hiddenSection).removeClass("js-hidden");
      $(hiddenSection).attr("aria-hidden", false);
      $(hiddenSection).css("display", "inline-block");
    }
    else if (expandHiddenSectionState == "true") {
      $(this).attr("data-expanded", false);
      $(this).attr("aria-expanded", false);
      $(hiddenSection).hide();
      $(hiddenSection).addClass("js-hidden");
      $(hiddenSection).attr("aria-hidden", true);
      $(hiddenSection).css("display", "none");
    }

  });

});

When to use this component

Use this component when a user needs to download a file as part of their journey.

When not to use this component

Do not use this component where the file is elsewhere on the web already. If that is the case, just a link to it would be enough.

How it works

Use clear, descriptive file names to help the user understand what they are downloading.

Include file size and type to reassure users that the file can be successfully downloaded and opened.

Include published date to ensure users can choose the most up to date information.

Use with other components

This component can be paired with components such as tags to help them identify the most up to date files.

Let users request another format

We must let users request another file format if they are using assistive technology. Our component does this using a link through the query form.

Research on this component

Downloads are stored, printed and circulated during a number of user processes. A clearly visible download option gives confidence to the user that information can be stored locally as a contingency.

User testing has found that some users do not understand what an ODS file format is. GDS recommends using this file format as the default. We added links, hover text and explanatory text to help users understand what it means.
