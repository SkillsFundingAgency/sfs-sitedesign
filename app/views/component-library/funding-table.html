Component library Funding table

A custom styled table used to display funding data in a clear and understandable format.

View more general GDS guidelines on this component in the GOV.UK design system.

The funding table component is used as the main way to display data on MYESF and view latest funding.

When to use this component

Use a funding table to help simplify complex funding data. Related data points can be arranged in sequential order. This makes it easier to work out formulas without having to display calculations.

This should only be used if the user can easily work out the relationship between each of the data points.

When not to use this component

Users may need the full picture to understand the data in the tables. The tables must not have missing data, otherwise the format becomes confusing and unfamiliar.

Do not list unrelated data in one table. Use headings or separate tables to display this information.

How it works

The tables are similar to the GOV.UK design system tables. There are some custom styles on the tables to make sure that the table contents are read out by screen readers.

The rules for displaying variance are:

  • only show variance if a figure has either increased or decreased from the previous figure
  • when it has increased, include an increase icon with the data
  • when it has decreased, include an decrease icon with the data
  • always display variance directly below the data it relates to

Three or more columns

This type of funding table layout is more suitable for displaying more complex funding data. When this type of table reaches a smaller resolution, it will wrap all centre columns (e.g. not first or last columns) so they display within the first column.

This means on smaller devices even the most complex funding tables will still be accessible and understandable without impacting on the overall experience for all of our users' needs (e.g. reduced font sizes).

The current rules that dictate adding a link are:

  • only offer a link to users when the related data is decimalised
  • this link should always appear directly below the descriptive funding table cell descriptive label (e.g. always the left hand side table cell in the first column)

Research on this component

Response to this component from users has been positive, with a number citing legibility as a benefit of viewing data in this format.

Users' ability to effectively interpret data from this component decreases however with additional columns. Consistent with usability research literature, it is important to users that only data pertinent to their needs is displayed.
