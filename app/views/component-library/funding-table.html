Component library Funding table

A custom styled table used to display funding data in a clear and understandable format.

View more general guidelines on this component in the GOV.UK design system.

The funding table component is used as the main way to display funding data on view latest funding.

When to use this component

Use a funding table to help simplify complex funding data. Related data points can be arranged to show their formulas without having to display calculations.

This should only be used if the user can easily work out the relationship between each of the data points.

When not to use this component

User may only be able to understand the data in the tables if they have the full picture. The tables must not have missing data otherwise the format becomes confusing and unfamiliar.

Do not list unrelated data in one table. Use headings or separate tables to display this information.

How it works

The tables are similar to the GOV.UK design system tables, there are some custom styles on the tables to ensure that the table contents are read out by screen readers.

In view latest funding, when the same column in different tables have the same heading, only the first heading is displayed. For example, see the 'Current total' column. This reduces duplication of content and clutter. Screen readers will still read out the heading for each table.

Two columns

This is the default design type of this component to use for displaying funding data in a table to end users in MyESF. Use this table type whenever possible for displaying funding data to users in the service.

No repeating column headers

Our version of the traditional GOV.UK design system table differs only in the use of column headers. We remove any duplicate funding table headers (e.g. 'Total deduction') where possible to reduce the cognitive load on our users and make the data more easily understandable.

The rules that dictate the removal of duplicate column headers are:

  • only visually display the first instance of a column header
  • use the <span class="visuallyhidden"> HTML markup to visually hide any duplicate column headers so that screen readers can still identify the data relationships

The current rules that dictate adding a link are:

  • only offer a link to users when the related data is decimalised
  • this link should always appear directly below the descriptive funding table cell descriptive label (e.g. always the left hand side table cell in the first column)

Research on this component

Response to this component from users has been positive, with a number citing legibility as a benefit of viewing data in this format.

Users' ability to effectively interpret data from this component decreases however with additional columns. Consistent with usability research literature, it is important to users that only data pertinent to their needs is displayed.
