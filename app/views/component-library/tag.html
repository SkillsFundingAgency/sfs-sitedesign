Component library Tag

A box of text, highlighted in a specific colour to indicate the status of something.

View more general guidelines on this component in the GOV.UK design system.

Tags are used across almost all of our services. They can be found on document exchange and view latest funding.

When to use this component

This component can be used to give the user a sense of when something has changed or there is something new to see. For example, using a “latest” or “not latest” tag indicates if the user is viewing the most up-to-date information or not.

The component can also be used to give a sense of what has been actioned on the page. On document exchange we use the tags to show if a file has been “downloaded” or “updated” by the user.

When not to use this component

Tags should be used sparsely to ensure they are prominent on the page.

Avoid using the tags for more than one purpose on the same page or service as this could confuse the user.

Do not use colours which would not normally be related to the message it contains.

How it works

MYESF use seven different types of tag which span three colour schemes - red, blue and grey.

Red

There are two red coloured tag types which are reserved for important notifications and user actions. These are labelled as:

  • 'New' - use this tag to inform users that they have received something from the ESFA.
  • 'Updated' - use this tag to inform users that there has been a change in a publication or allocation.

Blue

There are three blue coloured tag types which are reserved for standard informational purposes. These are labelled as:

  • 'Downloaded' - use this tag to inform local authority type parent users that one of their child organisations has both received something new from the ESFA and also opened or downloaded it
  • 'Latest' - use this tag to inform users that something they are viewing is the latest version available
  • 'Received' - use this tag to inform local authority type parent users that one of their child organisations has received something new from the ESFA

Grey

There are two grey coloured tag types which are reserved for low importance informational purposes. These are labelled as:

  • 'Final' - use this tag when informing users that something they are viewing is not the latest version available plus is also the final publication from the ESFA for that period
  • 'Not latest' - use this tag when signifying to users that something they are viewing is not the latest version available

Content is important in tags, the tags should clearly convey their message using one or two words.

The tags must make sense even if the colour were to be removed, this ensures users with colour blindness or visual impairments still receive the message.

The tags should be marked up to be read out by screen readers for users with visual impairments.

Research on this component

The tag component has been comprehensively demonstrated during testing to satisfy key signposting needs. The tag has been quickly identified and deciphered by the vast majority of users, guiding them to recent changes or additions to the service that may require their attention.

There is currently a gap in research in how closely users associate a particular colour with a label, and how easily users can discern between labels of the same colour, especially with the blue tag, which has four possible labels. What evidence we have from research tells us that this issue results in limited misinterpretation.
