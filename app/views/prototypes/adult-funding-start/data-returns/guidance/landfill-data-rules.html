How to prepare your data file

  1. Your data file should be a comma-separated values (CSV) file. You can use spreadsheet software such as Excel to create a CSV file. Or you can set up your data management systems to produce an automated export of data in the correct format.
  2. Include column headings as the first row. Use the correct column heading shown in the data rules. The columns can be in any order. The column heading tells our system what data is in that column.
  3. List each reading or measurement in its own row.
  4. Check the data rules to make sure you have the correct data in each column. You can use the service to check the files before you send them.
  5. You can use any filename provided it ends in '.csv'. You may find it helpful to use a descriptive filename.

If you wish to, you can send data for more than one permit type in the same file. The maximum file size allowed is {{config.csv.maxFileSizeMb}}MB. Split the files if they are larger than this.

Exceptions and special cases

There are some exceptions where you may not have to fill mandatory fields because of circumstances, or special cases where your data needs to be adapted. The table below describes these exceptions and explains how you must format your data in these circumstances. It also gives example data.

When you need to give a ‘No result’ measurement

You should leave the Value field blank and enter ‘No result’ in the Txt_Value field if you’re reporting on a parameter but can’t give a measurement, for example, because a borehole is blocked. You should also give an explanation in the Comments field.

You must give a line of data for each parameter, including instances where you couldn’t take a measurement.

EA_ID Site_Name Rtn_Period Rtn_Type Mon_Point Mon_Date Parameter Qualifier Ref_Period Value Txt_Value Unit Meth_Stand Comments
AB1234YZ Greendale Landfill Qtr 4 2016 Landfill gas borehole Borehole 1 2016-01-01T09:14 Methane No result Borehole is blocked

When Ref_Period covers multiple measurements in a single calculation

Give date and time data from the end of a period if you’re reporting a Ref_Period measurement calculated across multiple readings such as ‘Average’, ‘Maximum daily’ or ‘Minimum daily’. In the example below the single date and time entered for a Ref_Period of ‘Hourly mean’ represents a set of readings taken from 3pm to 4pm on 1 August 2016.

EA_ID Site_Name Rtn_Period Rtn_Type Mon_Point Mon_Date Parameter Qualifier Ref_Period Value Txt_Value Unit Meth_Stand Comments
AB1234YZ Greendale Landfill Qtr 4 2016 Landfill gas engine Eng1 2016-08-01T16:00:00 NOX Hourly mean 380 mg/m3

When you need to report a descriptive measurement

You may need to report on parameters, for example ‘Weather conditions’, that have no numerical result so you should give a description instead. To do this leave the Value and Units fields blank and enter ‘See comment’ in Txt_Value. Then you must enter a description in the Comments field.
